This is a guide on setting up the AWS DevOps Agent with AWS CLI

This is a link to the AWS documentation on the AWS CLI API for the DevOps Agent: DevOps Agent API

Commands to set up the agent

To set up the agent you need to create a space, associate the relevant services to the space and enable the operator app for users to interact with the agent. You will also need to set up the relevant permissions for the agent space and the operator app.

Create Space

Command:

        
            bash

            aws devops-agent create-agent-space \
            --name <value> \
            --description <value> \
            --region <value>

Link

Associate AWS account - main account where the space is created

Command:

        
            bash

            aws devops-agent associate-service \
            --agent-space-id <value> \
            --service-id aws \
            --region <value> \
            --configuration '{
                "aws": {
                    "assumableRoleArn": <value>,
                    "accountId": <value>,
                    "accountType": "monitor"
                }
            }'
Enable the OperatorApp to access the given AgentSpace

Command:

        
            bash
            aws devops-agent enable-operator-app \
            --agent-space-id <value>
            --auth-flow <value>
            --operator-app-role-arn <value>

Arguments:
     --agent-space-id - the id of the agent space
     --auth-flow - the authentication flow configured for the OperatorApp; possible values are [iam|idc|idp]
     --operator-app-role-arn - the arn of the iam role we want to use for the OperatorApp

Associate external AWS account as a data source

Command:

        
            bash
            aws devops-agent associate-service \
            --agent-space-id <value> \
            --service-id <value> \
            --configuration <value> \
            --region <value>

Arguments:
     --agent-space-id - the id of the AgentSpace
     --service-id - the unique identifier of the service: aws
     --configuration - the configuration that directs how AgentSpace interacts with the given service:
     {
         "sourceAws": {
             "accountId": "123456789123",
             "accountType": "source",
             "assumableRoleArn": "cross-account-role-arn"
         }
     }
